River Corridor — Implementation Plan
What it is: A procedurally generated, impassable-to-towers corridor that runs from the left or right map edge to the base. Generates once at map load after trees. Pure placement constraint — no pathfinding cost difference, no gameplay effect on zombies.

Step 1 — TerrainManager: generate the river
Add a river_tiles: Dictionary = {} to TerrainManager, same pattern as obstacle_tiles and slow_tiles.
Add a _generate_river() function called in initialise() after _generate_obstacles() and before _validate_paths(). The generation logic:

Pick a random starting tile on the left edge (x=0) or right edge (x=GRID_COLS-1), somewhere between y=8 and y=GRID_ROWS-8 to avoid corners
Target tile is BASE_TILE
Use a drunkard's walk from start to base — each step biases toward the base but has randomness. Suggested weights: 60% move toward base, 20% move perpendicular up, 20% move perpendicular down
At each step, roll for width: 80% chance single tile, 20% chance expand to 2 tiles (add the tile directly above or below current position)
When widening, hold the 2-tile width for 2-4 steps before returning to 1, so it feels gradual not jagged
After the walk is complete, for every river tile check all 8 neighbours — if any neighbour is an obstacle tile, remove that obstacle tile from obstacle_tiles and call PathfindingManager.remove_obstacle() on it. This enforces the 1-tile clearance from trees.
Store all river tiles in river_tiles dictionary

Step 2 — TerrainManager: paint the river
Add _paint_river() called after _paint_obstacles() in initialise().

For each tile in river_tiles, place a ColorRect at the correct world position
Color: Color(0.72, 0.65, 0.45, 1.0) — a sandy/pale dirt tone that reads clearly against the green grass
Size 16×16, positioned at tile_to_world(cell) - Vector2(8, 8)
Add to _obstacles_container so it's managed in the same place as other terrain visuals

Step 3 — TerrainManager: expose a helper
Add:
gdscriptfunc is_river_tile(cell: Vector2i) -> bool:
    return river_tiles.has(cell)
Step 4 — game_map.gd: block tower placement on river tiles
In _is_tile_valid(), add one line alongside the existing obstacle and base tile checks:
gdscriptif TerrainManager.is_river_tile(tile_pos):
    return false
Step 5 — Validate and test

Confirm the river always connects edge to base with no gaps
Confirm tower placement is rejected on river tiles (red ghost, no placement)
Confirm zombies pathfind through river tiles normally
Confirm no river tile sits directly adjacent to a tree (clearance rule)
Confirm the river looks visually distinct and readable at game scale
Run 5-10 map generations and check the river path feels varied and organic each time
